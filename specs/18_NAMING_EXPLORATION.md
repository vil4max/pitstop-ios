# Product Naming Exploration

**Status:** Open investigation\
**Current working name:** PitStop

## Naming problem

The product concept has expanded from a service reminder into:

> a smart memory for your car.

The name should be evaluated against the actual product, not the
original maintenance-only concept.

## Current name: PitStop

Strengths: - short; - memorable; - automotive association is
immediate; - works for maintenance/service planning; - easy to say in
English; - visually brandable.

Weaknesses: - implies a service stop more than memory/history/notes; -
motorsport association can pull design toward aggressive automotive
language; - likely crowded as a generic automotive term; - trademark/App
Store/domain availability must be investigated.

Current recommendation:

> Keep PitStop as the working name during domain and beta development.

Do not rename before the product proves whether its primary retained
value is maintenance planning or car memory.

## Naming territories

### Memory / companion

Concept: the app remembers the car with the owner.

Working candidates for investigation:

``` text
CarMemo
MotorMemo
RoadMemo
AutoMemo
CarMind
Motory
```

Risk: generic or artificial naming.

### Car history / life

``` text
Carlog
Roadlog
Motolog
DriveLog
CarStory
```

Risk: sounds like a passive logbook; weak on proactive status.

### Care / ownership

``` text
CarCare
MotorCare
DriveCare
Caremile
CarKeep
```

Risk: crowded category and generic trademarks.

### Service rhythm

``` text
PitStop
NextStop
Service Mile
MileMark
Roadmark
```

Risk: may over-index on maintenance.

### Friendly invented brand

A later naming sprint may explore short invented names with: - 2--3
syllables; - easy pronunciation for B1 English; - no ambiguous
spelling; - automotive association optional; - broad enough for memory +
maintenance.

Do not generate hundreds of random AI names. Use explicit naming
criteria.

## Naming scorecard

Every serious candidate is scored 1--5:

  Criterion                    Meaning
  ---------------------------- -------------------------------------------
  Concept fit                  smart memory + maintenance
  Memorability                 recalled after one exposure
  Pronunciation                easy in English/Russian/Ukrainian context
  Spelling                     can be typed after hearing
  Emotional tone               friendly, not aggressive
  Extensibility                works beyond one maintenance screen
  Search distinctiveness       not hopelessly generic
  Legal availability           trademark investigation
  App Store availability       naming collision investigation
  Domain/social availability   optional, not decisive for beta

## Naming decision gate

Do not rename on taste alone.

At day-30 beta review ask:

``` text
Why did you open the app?
What do you think the app is for?
How would you describe it to another driver?
```

If users primarily say:

``` text
service / maintenance / when to do ТО
```

PitStop remains strongly aligned.

If users primarily say:

``` text
it remembers everything about my car
notes / history / don't forget
```

run a formal naming sprint.

## Failure criterion

A rename is unsuccessful if: - it requires explanation; - spelling must
be repeated; - automotive relevance disappears completely; - it sounds
like an OBD/diagnostic product; - the project loses GitHub/LinkedIn
continuity without a clear product gain.
